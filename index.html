Mobile Device Management Kiosk

Project Summary

The device counterpart to the Enterprise Plaform MDM solution. It handles content management/distribution, user authentication by various means (including standard login of course), learning record synchronization and of course, device management. All customizable remotely.

Built using Unity and Android Studio for SkillsVR. In collaboration with frontend, backend and full stack developers (who built the server side of the MDM).

My Role

I took over as the lead developer for the SkillsVR Library (Mobile Device Management Kiosk), in around May 2024. I was responsible for the planning and execution of all technical aspects of this project from that stage on.
Some of my key work entailed:
  • Developing a system for synchronizing learner data to a server using REST APIs, gracefully handling internet connection states
  • Heavy use of Android Java for developing plugins for use in Unity
  • Developing and maintaining support for VR devices
  • Creating and reviewing tech designs
  • Creating a varied test suite containing unit tests, functional tests, regression tests, UI tests etc.
  • Implementing an automated build and test pipeline using Github actions and GameCI
  • Implementing a system for enforcing control over the guardian on Meta Quest devices without Quest provided APIs
  • Managing an Android developer, providing work and overseeing their growth within Unity
  • Developing, running, improving functionality and providing support for remote updates of the software
  • Creating automations to streamline QA testing
  • Extensive work with Unity UGUI
  • Use of Unity XR Interaction Toolkit
  • Developing a way to securely apply wifi configurations to a device across device group and individual device configurations

Site Safe Foundation Passport

Project Summary

VR game meant to teach learners how to stay safe on worksites. The application is split into two parts, learning and assessment, the first teaches the learner how to be safe on worksites, while the second tests the learners knowledge and, should they pass, awards them with a Site Safe Foundation Passport.

My Role

Built the game's diorama (select correct order of events), hazard selection and PPE selection systems. Made heavy use of prefabs to reduce the need to rebuild and redo changes across scenes, and Unity's XR Interaction Toolkit for player interactions.

Kiwifruit Pruning

Project Summary

VR game aimed at those looking to get into kiwifruit pruning. This game teaches learners the skills needed to succesfully prune a kiwifruit orchard in both summer and winter.

My Role

A lot of time was put into getting the Quaternion math to work just right with Unity's XR systems. I also collaborated with a UI Designer to create a scoreboard for the game and implemented a combined implementation of a soft body vine system with hard "cut points" where mesh cutting could take place.

Devil's Punishment

Project Summary

Devil's Punishment was a sci-fi/post-apocalyptic survival horror 3D game with FPS (First person shooter) mechanics, where a group of prisoners are left behind in an abandoned underground facility. The game was never fully released to my knowledge.

As this was one of my first freelance projects, I was unpaid, working only for the right to use the work on my portfolio

My Role

I developed the game's room and prop generation, which involved procedurally generating a room layout, splitting the room into a grid and then generating various prop clusters (usually around a central prop) at those grid locations depending on the prop type. The generation system also ensured the room had enough walk room for the player to reach all grid points in the room.

MDM Provisioning Tool

Project Summary

The windows based provisioning tool used to install and setup the MDM Kiosk on devices, and register devices on an external server. Makes extensive use of RESTAPIs for logins and fetching data from server. Adb is used to run commands on devices and pass intent data to a special receiving application on target devices, as well as to fetch various device information and pass it to the server for storage/processing. Built using Unity for SkillsVR. In collaboration with .NET developers (who built the server which the tool communicates with), a UI Designer and Product Owner.

My Role

I took over as the lead developer for the SkillsVR Provisioning Tool, in around May 2024. I was responsible for the planning and execution of all technical aspects of this project from that stage on. Some of my key work entailed:
  • Created a searchable dropdown that updates as you type
  • Created a system for users to install applications to their devices from a server
  • Maintained device to server enrollment as enrollment requirements evolved, including heavy use of REST APIs and asynchronous programming with both Unity Coroutines and C# Tasks
  • Consulted with a UI Designer while implementing their UI designs via Unity UGUI, ensuring dynamic support for a variety of screen resolutions
  • Coached other developers and conducted code review to ensure both developer growth within the project and Unity, as well as stable, timely updates
  • Created various tech designs in collaboration with our backend team
